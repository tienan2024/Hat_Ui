**变更类型:** 功能增强

**改动概述:** 在热成像画面上**右键 → 详情窗口**，弹出一个独立大窗口，同时显示热成像画面、温度仪表盘和热区信息。

**改动详情:**

| 文件 | 改动说明 |
|------|----------|
| `hat_main.py` | 新增 `ThermalPopWindow` 类（独立窗口，含热成像 + 3个 RingGauge + 热区信息）；修改 `ThermalWidget._on_right_click` 新增"详情窗口"菜单项；新增 `_show_thermal_pop_window()` 方法 |

**使用方式:**

1. 在热成像画面上**右键**

3. 弹出 800×720 独立窗口，显示：

## 2026-06-03

### 修复：远程工控机模式下连接按钮误断开工控机 TCP 连接

**变更类型:** Bug 修复

**问题描述:** 当用户连接远程工控机后（右键菜单 → 连接远程工控机），选择工控机上的串口并点击"连接"按钮时，系统会错误地断开工控机的 TCP 连接，而不是打开所选的串口。

**根本原因:** `_ext_on_mcu_connect()` 方法在远程模式下，`is_connected()` 返回 True（TCP 已连接）时，错误地调用了 `disconnect()` 断开 TCP，而不是调用 `open_remote_port()` 打开串口。

**修复内容:**

| 文件 | 改动说明 |
|------|----------|
| `hat_main.py` 2496-2572 | 重写 `_ext_on_mcu_connect()`，区分远程模式的"打开串口"和"断开串口"两种操作 |
| `hat_main.py` 2649 | `_ext_connect_remote_dialog()` 中建立 TCP 连接后重置 `_remote_port_opened` 标记 |
| `hat_main.py` 2660 | `_ext_disconnect_remote()` 断开 TCP 前重置 `_remote_port_opened` 标记 |

**修复后的远程模式行为:**

1. 右键菜单 → 连接远程工控机 → 建立 TCP 连接
2. 串口下拉框自动刷新为工控机上的串口列表
3. 选择串口 → 点击"连接"按钮 → 调用 `open_remote_port()` 打开串口
4. 再次点击"连接"按钮（此时显示"断开"）→ 关闭串口但保持 TCP 连接
5. 通过右键菜单 → 断开远程连接 → 完全断开 TCP

### 新增：Lepton 热像仪串口转发支持（通过工控机 relay）

**变更类型:** 功能增强

**改动概述:** Lepton 热像仪现在也支持通过工控机 relay 转发数据，与 MCU relay 类似的远程访问能力。

**改动详情:**

| 文件 | 改动说明 |
|------|----------|
| `改造参考/relay/relay-server.js` | 扩展为同时支持 MCU（端口 6001）和 Lepton（端口 6002）双串口独立运行 |
| `改造参考/relay/config.json` | 新增 `lepton` 配置段：enabled, port, baudRate, tcpPort |
| `hat_extensions.py` | `ThermalLeptonReader` 新增远程 TCP 模式（`open_remote()`, `list_remote_ports()`, `open_remote_port()` 等）；`ThermalCameraManager` 新增 `connect_lepton_remote()` 方法和 `lepton_remote` 模式 |
| `hat_main.py` | 热像仪设备下拉框新增"远程 Lepton (中继)"选项；`thermal_state_label` 添加右键菜单；新增 `_ext_lepton_context_menu()`、`_ext_connect_lepton_remote_dialog()`、`_ext_disconnect_lepton_remote()` 回调 |
| `config.json` | 新增 `lepton_relay` 配置段（host/port 默认值） |

**使用方式:**

1. 工控机运行 `relay-server.js`（需确保 `lepton.enabled=true`）
2. PC 上选择热像仪模式为"远程 Lepton (中继)"
3. 右键状态标签 → 连接远程工控机 → 输入工控机 IP 和端口（默认 6002）
4. TCP 连接成功后，串口下拉框自动刷新为工控机上的串口列表
5. 选择 Lepton 连接的串口 → 点击"连接"按钮打开串口
6. 热成像画面开始显示

### 新增：可见光摄像头视频流转发支持（通过工控机 relay）

**变更类型:** 功能增强

**改动概述:** 可见光 USB 摄像头接在工控机上，通过 Python/OpenCV + HTTP MJPEG 转发到上位机。

**改动详情:**

| 文件 | 改动说明 |
|------|----------|
| `改造参考/relay/relay-server.js` | 新增视频流服务：启动 Python video-streamer.py（OpenCV 后端 700/VFW），输出 HTTP MJPEG 流（端口 8080） |
| `改造参考/relay/video-streamer.py` | 新增 Python MJPEG HTTP 流服务器，自动尝试 CAP_DSHOW → CAP_MSMF → CAP_ANY 后端 |
| `改造参考/relay/config.json` | 新增 `video` 配置段：enabled, device, width, height, fps, port |
| `hat_main.py` | 网络摄像头 URL 从 config.json 读取（`camera.network_url`），支持配置工控机视频流地址 |
| `config.json` | 新增 `video_relay` 配置段（host/port 默认值）；`camera.network_url` 可配置为工控机视频流地址 |

**工控机部署:**

1. 无需安装 FFmpeg，使用 Python/OpenCV 原生支持
2. 修改 `relay/config.json`：

   ```json
   "video": {
     "enabled": true,
     "device": "USB Camera",
     "width": 640,
     "height": 480,
     "fps": 15,
     "port": 8080
   }
   ```

3. 运行 `node relay-server.js`（会自动启动 Python video-streamer.py）

**注意：** Windows 上推荐分辨率 640x480（VFW 后端 700 对高分辨率支持不稳定）

**上位机使用:**

1. 修改 PC 上 `config.json` 的 `camera.network_url` 为工控机视频流地址：

   ```json
   "camera": {
     "network_url": "http://192.168.3.100:8080/stream"
   }
   ```

2. 在 UI 上选择"网络摄像头"模式即可显示工控机摄像头画面

## 2026-06-02

### 温度仪表盘 3 级报警系统

**变更类型:** 功能增强

**改动概述:** 将原有的单阈值开关式报警改为 3 级报警，支持黄色预警闪烁和红色报警闪烁，阈值可通过右键菜单修改。

**改动详情:**

| 模块 | 文件 | 改动说明 |
|------|------|----------|
| 常量 | `hat_main.py:167-169` | 阈值常量从 4 级简化为 3 级: `TEMP_WARN_LEVEL_1=40℃`, `TEMP_WARN_LEVEL_2=70℃` |
| 报警状态 | `hat_main.py:303-309` | `temperature_alarm_state()` 改为 3 级返回，新增动态阈值参数 |
| 仪表盘控件 | `hat_main.py:1260-1316` | `RingGauge` 新增 `set_alarm_level(0/1/2)` 方法，保留 `set_alarm()` 兼容旧接口 |
| 主窗口初始化 | `hat_main.py:1731-1733` | 新增 `self.threshold_warn` 变量 (默认 40℃) |
| 报警逻辑 | `hat_main.py:2284-2309` | `update_temperature_alarm()` 使用 3 级逻辑驱动所有仪表盘 |
| 右键菜单 | `hat_main.py:2695-2741` | 对话框支持同时修改预警(黄闪)和报警(红闪)两个阈值，含校验 |

**3 级报警规则:**

| 温度范围 | 仪表盘颜色 | 级别 |
|----------|------------|------|
| < 预警阈值 (默认 40℃) | 绿色 | 正常 |
| ≥ 预警阈值 且 < 报警阈值 | 黄色闪烁 | 二级预警 |
| ≥ 报警阈值 (默认 70℃) | 红色闪烁 | 三级报警 |

**接口变更:**

- `temperature_alarm_state(temp)` → `temperature_alarm_state(temp, warn_threshold=None, alarm_threshold=None)`
- `RingGauge` 新增 `set_alarm_level(level)` — level: 0=正常, 1=黄闪, 2=红闪
- `RingGauge.set_alarm(bool)` 保留兼容 — True=级别2, False=级别0

**向下兼容:**

- `set_alarm()` 旧接口仍然可用，内部转发到 `set_alarm_level()`
- `TEMP_WARN_LEVEL_1` / `TEMP_WARN_LEVEL_2` 全局常量仍存在，作为默认值
