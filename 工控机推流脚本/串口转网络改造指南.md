# 串口转网络改造指南

最后更新：2026-06-01

> **说明**：`relay/` 中继服务是**协议无关**的通用桥接程序。多个项目可共用同一套 relay 代码，只需修改 `config.json` 中的串口号和波特率。relay 不解析也不过滤任何 MCU 数据帧，仅做透明转发。

## 适用场景

原有项目中，下位机（MCU）通过串口直连前端电脑（Node.js 服务器）。现需要将串口链路替换为：

```
改造前:  MCU ──串口──→ PC (Node.js)
改造后:  MCU ──串口──→ 工控机 ──WiFi/TCP──→ PC (Node.js)
```

工控机运行 Node.js 中继服务，PC 端通过 TCP 连接工控机获取数据。

---

## 改造总览

需要改动 3 个层面：

| 层面 | 改动内容 | 涉及文件 |
|------|----------|----------|
| 工控机端 | 新建中继服务（串口↔TCP桥接） | 新建 `relay/` 目录 |
| 后端 | 串口管理器增加远程 TCP 连接能力 | `serial-manager.js`、`index.js` |
| 前端 | 增加连接模式切换 UI | `App.jsx`、`Controls.jsx` |

## 第一步：创建工控机中继服务

在项目根目录创建 `relay/` 目录，包含以下文件：

### relay/package.json

```json
{
  "name": "sensor-relay",
  "version": "1.0.0",
  "type": "module",
  "description": "传感器中继服务 - 串口(MCU) <-> TCP(PC) 桥接",
  "main": "relay-server.js",
  "scripts": {
    "start": "node relay-server.js"
  },
  "dependencies": {
    "serialport": "^13.0.0"
  }
}
```

### relay/config.json

```json
{
  "serial": {
    "port": "COM3",
    "baudRate": 115200
  },
  "tcp": {
    "port": 6001
  }
}
```

- `serial.port`：工控机连接 MCU 的串口号
- `serial.baudRate`：与 MCU 一致的波特率
- `tcp.port`：TCP 监听端口，PC 连接此端口

### relay/relay-server.js

核心逻辑，完整代码参考本项目的 `relay/relay-server.js`。关键设计：

```
┌─────────────────────────────────────────┐
│           relay-server.js               │
│                                         │
│  ┌──────────┐    ┌───────────────────┐  │
│  │ Serial   │    │ TCP Server        │  │
│  │ (to MCU) │    │ (for PC to conn)  │  │
│  │          │    │                   │  │
│  │ 收到数据  │──→│ 转发给已连接的PC  │  │
│  │          │    │                   │  │
│  │ 发送指令  │←──│ 收到PC的指令      │  │
│  └──────────┘    └───────────────────┘  │
│                                         │
│  心跳检测: 30秒无响应断开PC             │
│  串口重连: 3秒自动重连                  │
│  TCP断连: 自动清理，等待新连接          │
└─────────────────────────────────────────┘
```

**核心代码结构：**

```javascript
import { SerialPort } from 'serialport';
import { createServer } from 'node:net';

let serialPort = null;
let tcpClient = null;
let tcpServer = null;
let pingTimer = null;

// ============ 串口侧 ============

function openSerial(portPath) {
  // 防止重复打开
  if (serialPort) {
    serialPort.removeAllListeners();
    if (serialPort.isOpen) {
      serialPort.close(() => { serialPort = null; doOpenSerial(); });
      return;
    }
    serialPort = null;
  }
  doOpenSerial();
}

function doOpenSerial() {
  const port = new SerialPort({
    path: currentSerialPath,
    baudRate: BAUD_RATE,
    dataBits: 8, parity: 'none', stopBits: 1,
    autoOpen: false
  });
  serialPort = port;

  port.on('data', (data) => {
    // 收到MCU数据，按行分割，全部透传给PC（协议无关）
    buffer += data.toString();
    const lines = buffer.split('\r\n');
    buffer = lines.pop();
    for (const line of lines) {
      if (line.length > 0) sendToPC(line + '\r\n');
    }
  });

  port.on('error', (err) => { /* 日志 */ });

  port.on('close', () => {
    serialPort = null;
    sendToPC('$STATUS,DISCONNECTED\r\n');  // 通知PC
    setTimeout(openSerial, 3000);           // 3秒重连
  });

  port.open((err) => {
    if (err) { setTimeout(openSerial, 3000); return; }
    sendToPC('$STATUS,CONNECTED\r\n');      // 通知PC
  });
}

// ============ TCP 侧 ============

function cleanupTcpClient(reason) {
  if (tcpClient) {
    tcpClient.removeAllListeners();
    tcpClient.destroy();
    tcpClient = null;
  }
  clearPingTimer();
}

function startTCPServer() {
  if (tcpServer) { try { tcpServer.close(); } catch {} }

  const server = createServer({ allowHalfOpen: false }, (socket) => {
    cleanupTcpClient('new connection');
    tcpClient = socket;
    socket.setKeepAlive(true, 10000);
    socket.setNoDelay(true);
    resetPingTimer();

    socket.on('data', (data) => {
      // 收到PC数据，解析行
      const lines = data.toString().split('\r\n');
      for (const line of lines) {
        if (line.length === 0) continue;
        if (line.startsWith('$CMD,')) {
          handleCommand(line);  // 控制命令（心跳、串口列表等）
        } else {
          sendToMCU(line + '\r\n');  // 其他 $ 帧全部透传给 MCU
        }
      }
    });
    socket.on('error', () => cleanupTcpClient('error'));
    socket.on('close', () => cleanupTcpClient('close'));
  });

  server.on('error', (err) => {
    if (err.code === 'EADDRINUSE') setTimeout(startTCPServer, 3000);
  });

  server.listen(TCP_PORT);
  tcpServer = server;
}

// 启动
openSerial();
startTCPServer();
```

**必须处理的断连场景：**

| 场景 | 处理方式 |
|------|----------|
| MCU 串口断开 | `port.on('close')` → 置空 → 3秒重连 → 通知 PC |
| PC TCP 断开 | `socket.on('close')` → `cleanupTcpClient()` → 等待新连接 |
| PC TCP 异常 | `socket.on('error')` → `cleanupTcpClient()` |
| 新 PC 连接（替换旧的） | 先 `cleanupTcpClient()` 再接受新连接 |
| TCP server 端口占用 | `EADDRINUSE` → 3秒后重建 server |
| 心跳超时 | 30秒无 `$PING` → `cleanupTcpClient()` |

## 第二步：改造后端串口管理器

### serial-manager.js 改动

**新增远程 TCP 连接方法 `openRemote(host, port)`：**

```javascript
import { Socket } from 'node:net';

// 在原有 SerialManager 类中新增：

async openRemote(host, port = 6001) {
  // 1. 强制清理旧连接
  this._forceCleanupTcp();

  // 2. 创建 TCP socket 连接工控机
  const socket = new Socket();
  socket.setKeepAlive(true, 10000);
  socket.setNoDelay(true);
  socket.setTimeout(5000);

  // 3. 数据处理复用原有逻辑
  let buffer = '';
  socket.on('data', (data) => {
    buffer += data.toString();
    const lines = buffer.split('\r\n');
    buffer = lines.pop();
    for (const line of lines) {
      this._processLine(line);  // 复用原有 $DATA/$ACK 解析
    }
  });

  // 4. 断连处理
  socket.on('close', () => {
    this.connected = false;
    this.emit('disconnect');
    this._scheduleReconnect();  // 3秒自动重连
  });

  // 5. 连接
  await new Promise((resolve, reject) => {
    socket.connect(port, host, () => {
      socket.setTimeout(0);
      resolve();
    });
  });

  this.connected = true;
  this.mode = 'remote';
  this.emit('connect');
}

_forceCleanupTcp() {
  if (this.reconnectTimer) {
    clearTimeout(this.reconnectTimer);
    this.reconnectTimer = null;
  }
  if (this.tcpSocket) {
    this.tcpSocket.removeAllListeners();
    this.tcpSocket.destroy();
    this.tcpSocket = null;
  }
  this.connected = false;
  this.mode = null;
}
```

**关键：`_processLine()` 必须本地串口和远程 TCP 共用**

```javascript
_processLine(line) {
  if (line.startsWith('$ACK,')) {
    this._handleAck(line);        // 原有逻辑
  } else if (line.startsWith('$DATA,')) {
    this._handleSensorData(line); // 原有逻辑
  } else if (line.startsWith('$STATUS,')) {
    // 工控机转发的MCU状态
    this.emit(line.includes('CONNECTED') ? 'mcu_connect' : 'mcu_disconnect');
  }
}
```

**`_writeData()` 根据模式选择输出：**

```javascript
_writeData(msg, callback) {
  if (this.mode === 'local' && this.port?.isOpen) {
    this.port.write(msg, callback);
  } else if (this.mode === 'remote' && this.tcpSocket && !this.tcpSocket.destroyed) {
    this.tcpSocket.write(msg, callback);
  } else {
    callback(new Error('No available connection'));
  }
}
```

### index.js 改动

新增 HTTP API 端点：

```javascript
// 连接工控机
app.post('/api/serial/connect-remote', async (req, res) => {
  const { host, port } = req.body;
  await serialManager.openRemote(host, port || 6001);
  broadcast({
    type: 'status',
    serial: true,
    serialMode: 'remote',
    serialInfo: serialManager.getConnectionInfo()
  });
  res.json({ success: true });
});

// 查询工控机串口列表
app.get('/api/serial/remote-ports', async (req, res) => {
  const ports = await serialManager.listRemotePorts();
  res.json(ports);
});

// 切换工控机串口
app.post('/api/serial/remote-open', async (req, res) => {
  const { port } = req.body;
  await serialManager.openRemotePort(port);
  res.json({ success: true });
});

// 断开连接（本地和远程通用）
app.post('/api/serial/disconnect', (req, res) => {
  serialManager.close();
  broadcast({
    type: 'status',
    serial: false,
    serialMode: null,
    serialInfo: null
  });
  res.json({ success: true });
});
```

## 中继控制协议

PC 与工控机之间除了透传 MCU 数据帧，还有以下控制帧：

| 方向 | 帧 | 说明 |
|------|-----|------|
| PC→工控机 | `$PING\r\n` | 心跳检测 |
| 工控机→PC | `$PONG\r\n` | 心跳回复 |
| PC→工控机 | `$CMD,LIST_PORTS\r\n` | 查询串口列表 |
| 工控机→PC | `$PORTS,COM1,COM3,...\r\n` | 串口列表响应 |
| PC→工控机 | `$CMD,OPEN,COM3\r\n` | 切换串口 |
| 工控机→PC | `$CMD_OK,OPEN\r\n` | 切换成功 |
| 工控机→PC | `$CMD_ERR,OPEN,reason\r\n` | 切换失败 |
| 工控机→PC | `$STATUS,CONNECTED\r\n` | MCU 串口已连接 |
| 工控机→PC | `$STATUS,DISCONNECTED\r\n` | MCU 串口已断开 |

所有控制帧以 `$` 开头，与 MCU 数据帧格式一致，便于统一解析。

### 工控机

1. 安装 Node.js v18+
2. 复制 `relay/` 目录
3. `npm install --registry=https://registry.npmmirror.com`
4. 编辑 `config.json` 设置串口号和端口

1. 替换 `server/serial-manager.js`

3. 替换前端 `App.jsx`、`Controls.jsx`
4. 更新 `config.json` 添加 `sensor_relay` 段

## 多项目共用 relay

`relay/` 是协议无关的通用桥接服务，多个项目可共用同一套代码。

**共用方式**：将 `relay/` 目录复制到工控机，每个项目只需修改 `config.json`：

```json
{
  "serial": {
    "port": "COM3",        ← 改为该项目 MCU 连接的串口号
    "baudRate": 115200     ← 改为该项目的波特率
  },
  "tcp": {
    "port": 6001           ← 可保持不变
  }
}
```

`relay-server.js`、`package.json`、`README.md` **不需要任何修改**。

**工作原理**：relay 不解析也不过滤 MCU 数据帧，仅做透明转发。PC 端的 `serial-manager.js` 负责解析具体协议（`$DATA`、`$ACK` 等），relay 只管传字节流。

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 连接后收不到数据 | 工控机串口号错误 | 网页上点 ↻ 刷新串口列表，选正确的串口 |
| 重启后连不上 | 旧 TCP 连接未清理 | relay 和 PC 端都有自动清理，等 3 秒重试 |
| 数据延迟高 | 心跳/重连间隔太长 | 正常不应超过 100ms，检查 WiFi 信号 |
| 串口列表为空 | 工控机未接 MCU 或串口驱动缺失 | 检查设备管理器中的串口 |
